## 🔍 使用方式

### 第一步：收集原始素材

将真实的、有价值的素材保存到 `原始数据/` 目录：

**即刻动态**（强烈推荐）
- 导出你的即刻动态为CSV
- 保存为 `即刻动态.csv`
- 内容包括：你对产品、技术、现象的评价和吐槽

**个人笔记**
- 思考过程、洞察、总结
- 项目经验、学到的东西
- 真实遇到的问题和解决方案

**项目案例**
- 做过的项目、产品
- 成功或失败的经验
- 具体的数字、时间、结果

### 第二步：分类和索引

定期将原始素材分类到 `已分类/` 目录：

**按主题分类**
- AI工具 / 编程 / 产品开发 / 技术 / 等等
- 便于后续快速查找

**建立索引**
- 创建 `素材库索引.md`
- 按主题列出素材位置
- 方便Claude Code搜索

### 第三步：在写作中使用

写作时，Claude Code会：

**方法A：直接搜索原始数据（推荐）**
```
使用Grep工具在即刻动态.csv中搜索关键词
搜索词: "Claude Code|编程工具|效率"
返回相关的即刻内容
```

**方法B：查看已分类的素材**
```
打开 已分类/AI工具/Claude相关.md
查看已整理的素材
```

### 第四步：改写成文章内容

**重要**：不能直接复制粘贴！

- ❌ 即刻原文："Claude Code出了，用了两周，确实比Cursor好用"
- ✅ 改写后："我用了两周Claude Code，确实比Cursor好用。速度快不少，平均每个项目省3-5小时。"

**改写原则**：
- 将零散的即刻内容组织成长文逻辑
- 加入更多细节和论证
- 融入周围的上下文
- 使用你的写作风格

## ✨ 什么样的素材有价值？

### ✅ 有价值的素材

- **真实经历**：你实际做过、用过、经历过
- **具体数字**：不是"显著"，而是"快30%"
- **对比感悟**："vs之前的工具，这个更..."
- **问题和解决**：遇到了什么问题，怎么解决的
- **个人观点**：你真实的评价和立场，而不是模棱两可

### ❌ 没价值的素材

- **道听途说**：听别人说的，自己没验证过
- **模糊不清**："很多人说"、"据说"、"应该"
- **过度夸大**：吹嘘，不符合实际
- **没有细节**："很好用"、"提升不少"、"很重要"
- **AI写的内容**：之前用AI生成的内容（没有真实感）
